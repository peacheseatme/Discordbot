# Coffeecord Privacy Policy

**Last updated: March 2026**

Coffeecord respects user privacy and is designed to collect only the information necessary to provide its features.

## Information Collected

Coffeecord may store limited data required to operate its features.

This may include:

### Server Configuration Data

- Guild (server) IDs
- Channel IDs
- Role IDs
- Command settings
- Moderation configuration
- Leveling configuration
- Logging settings

### User Feature Data

- User IDs for leveling systems
- XP values
- Quest progress
- Reminder timers
- Moderation records (warnings, mutes, etc.)
- DM opt-out preference (if you use `/optout` to stop receiving staff DMs)

### Supporter Data

If a user supports Coffeecord through Ko-fi, the system may store:

- Discord user ID
- Supporter status
- Feature access permissions (such as animated level cards)

**No payment information is stored by Coffeecord.** Payments are handled entirely by Ko-fi.

### Message Content

Coffeecord does not permanently store message content unless required for a feature such as:

- Moderation logging
- Translation processing
- Command execution

Temporary processing may occur when commands are used.

## Data Usage

Stored data is used only for:

- Running bot features
- Maintaining server settings
- Providing leveling systems
- Processing moderation actions
- Enabling supporter perks

**Data is never sold or shared with third parties.**

## Data Storage

Data is stored securely on servers used to operate the bot.

Access is limited to the bot developer for maintenance and debugging purposes.

## Data Removal

Server administrators may remove stored data by:

- Removing the bot from the server
- Requesting deletion through the support server

Upon removal from a server, related configuration data may be deleted.

## Third-Party Services

Coffeecord may interact with third-party services including:

- Discord
- Ko-fi
- Translation APIs

These services operate under their own privacy policies.

## Changes to This Policy

This policy may be updated as features evolve. Continued use of the bot constitutes acceptance of the updated policy.
